A recently conducted docker push to Quay.io shows the wrong timestamp for one or more images

Your machine/VM's time has desynced

The timestamp we show for the Changed field is generated by Docker when the image is created. If the time on the machine on which the image was created is out of sync, the timestamp shown will be different on Quay.io as well. Usually this means a machine needs to be synced with NTP.
